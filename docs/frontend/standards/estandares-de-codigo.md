Índice de convenciones que aplican a todo el código frontend del equipo. El código que todos escribimos debe parecer escrito por una sola persona.

## Qué cubre esta sección

| Documento | Qué define |
|---|---|
| [Naming](./naming) | Convenciones de nombres para archivos, variables, funciones y componentes |
| [Tipado](./tipado) | TypeScript: interfaces, tipos, generics y cuándo usar cada uno |
| [Componentes](./componentes) | Anatomía, clasificación y reglas de composición |
| [Manejo de Estado](./manejo-de-estado) | Cuándo usar estado local, de servidor o global |
| [Integración de APIs](./integracion-apis) | Cómo consumimos APIs desde el frontend |
| [Jerarquía de Capas](./jerarquia-capas) | z-index, stacking context y orden visual |
| [Accesibilidad](./accesibilidad) | Reglas mínimas que todo dev debe cumplir |
| [Seguridad Frontend](./seguridad-frontend) | XSS, tokens, inputs y variables de entorno |

## La regla de oro

Si algo no está en este índice, la decisión por defecto es la más simple y la más legible. Cuando el equipo converge en una convención nueva, se documenta aquí.
