# ¿Qué es Git?

Git es el sistema de control de versiones moderno más utilizado en el mundo. Es un proyecto de código abierto desarrollado originalmente por Linus Torvalds en 2005.

## Características principales

### Arquitectura distribuida

Git es un DVCS (sistema de control de versiones distribuido). A diferencia de sistemas centralizados como SVN, cada desarrollador obtiene su propio repositorio local con el historial completo de confirmaciones.

### Rendimiento

Las operaciones básicas de Git están optimizadas:
- Confirmación de cambios
- Ramificación
- Fusión
- Comparación de versiones

Git se centra en el contenido del archivo, no en los nombres, permitiendo cambios de nombre, divisiones y reorganizaciones eficientes.

### Seguridad

Git protege la integridad del código fuente mediante algoritmos criptográficos. El contenido de archivos, relaciones entre directorios, versiones, etiquetas y confirmaciones están protegidos.

### Flexibilidad

Git está diseñado para soportar diversos flujos de trabajo sin imponer una forma determinada de trabajar.

## Ventajas de Git

- **Desarrollo distribuido**: Cada desarrollador tiene un repositorio completo
- **Flujo de trabajo de ramas**: Las ramas son baratas y fáciles de fusionar
- **Solicitudes de incorporación de cambios**: Facilitan la revisión de código
- **Comunidad**: Git se ha convertido en el estándar esperado para nuevos proyectos
- **Historial local**: No requiere conexión de red para crear confirmaciones

## Comparación con otros sistemas

### Git vs SVN

| Aspecto | Git | SVN |
|--------|-----|-----|
| Arquitectura | Distribuida | Centralizada |
| Repositorio local | Completo | Solo copia de trabajo |
| Confirmaciones | Locales primero | Remotas inmediatas |
| Ramas | Baratas y rápidas | Costosas |
| Historial | Completo localmente | En servidor |

## Casos de uso

Git es ideal para:
- Proyectos de código abierto
- Equipos distribuidos
- Desarrollo ágil
- Integración continua
- Cualquier proyecto que requiera control de versiones
