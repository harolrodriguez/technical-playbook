# Git Guide: Learning Resources

Esta sección contiene guías técnicas profundas sobre cómo usar Git. Aquí aprendes **cómo hacer las cosas**, no **qué debes hacer**.

## Diferencia entre Engineering Standards y Git Guide

| Aspecto | Engineering Standards | Git Guide |
|--------|----------------------|-----------|
| **Qué es** | Políticas y gobernanza | Recursos de aprendizaje |
| **Obligatorio** | Sí, todos deben cumplir | No, es opcional según necesidad |
| **Contenido** | Branching strategy, conventional commits, merge policies | Técnicas avanzadas, troubleshooting, configuración |
| **Ejemplo** | "Debes usar GitFlow" | "Así se resuelven conflictos con rebase" |

## Contenido de esta sección

- **Setup & Configuration**: SSH, GPG, configuración inicial
- **Merge Strategies**: Merge vs Rebase vs Squash, cuándo usar cada uno
- **Conflict Resolution**: Técnicas para resolver conflictos complejos
- **Advanced Operations**: Reset, Revert, Rebase, Cherry-pick
- **Inspection**: Advanced log, gitK, bisect
- **Git Hooks**: Pre-commit, pre-push, commit-msg hooks
- **Advanced Topics**: Submodules, Subtrees, Maintenance, Cleanup

## Cómo usar esta guía

1. **Si necesitas configurar Git**: Ve a [Setup & SSH](/knowledge-base/git-guide/setup-ssh)
2. **Si tienes un conflicto**: Ve a [Conflict Resolution](/knowledge-base/git-guide/conflict-resolution)
3. **Si quieres entender merge strategies**: Ve a [Merge Strategies](/knowledge-base/git-guide/merge-strategies)
4. **Si necesitas deshacer cambios**: Ve a [Reset, Revert, Rebase](/knowledge-base/git-guide/reset-revert-rebase)
