# Base de Conocimientos

Bienvenido a la Base de Conocimientos. Aquí encontrarás guías técnicas profundas, patrones de desarrollo y soluciones a problemas comunes.

A diferencia del [Handbook](/handbook/intro), que define **qué debes hacer**, la Base de Conocimientos te enseña **cómo hacerlo** y **por qué**.

## Secciones

### [Git Guide](/knowledge-base/git-guide)

Guías técnicas sobre Git: desde configuración básica hasta estrategias avanzadas de merge, resolución de conflictos y mantenimiento del repositorio.

**Cuándo leer**: Cuando necesites profundizar en Git más allá de lo básico.

### [Best Practices](/knowledge-base/best-practices/async-patterns)

Patrones y prácticas recomendadas para escribir código robusto: manejo de errores, gestión de estado, y patrones asincronos.

**Cuándo leer**: Cuando estés diseñando una solución y quieras evitar problemas comunes.

### [Troubleshooting](/knowledge-base/troubleshooting/common-issues)

Soluciones a problemas comunes que encontrarás durante el desarrollo, y técnicas de debugging.

**Cuándo leer**: Cuando algo no funciona y necesitas una solución rápida.

## Cómo usar esta sección

- **Busca por problema**: Si algo no funciona, busca en Troubleshooting
- **Aprende patrones**: Lee Best Practices para entender cómo hacer las cosas bien
- **Profundiza en Git**: Usa Git Guide cuando necesites ir más allá de lo básico

## Diferencia con el Handbook

| Handbook | Base de Conocimientos |
|----------|----------------------|
| Define estándares | Enseña técnicas |
| "Debes hacer esto" | "Aquí está cómo" |
| Obligatorio | Opcional |
| Corto y directo | Detallado y profundo |
| Políticas | Conocimiento |

**Ejemplo**:
- Handbook: "Usa Conventional Commits para todos los commits"
- Base de Conocimientos: "Aquí está cómo escribir buenos commits, qué herramientas usar, y cómo configurarlas"
