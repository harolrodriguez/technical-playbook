Formato, frecuencia y proceso para proponer una Tech Talk. Espacio para compartir aprendizajes, explorar tecnologías y crecer como equipo.

## Qué es una Tech Talk

Una Tech Talk es una presentación técnica de 20-40 minutos donde alguien del equipo comparte algo que aprendió, exploró o quiere discutir con el resto.

No es una clase magistral ni una presentación corporativa. Es una conversación técnica con estructura.

## Frecuencia

Una Tech Talk cada dos semanas, en un horario fijo acordado por el equipo. Si nadie propone tema, se cancela sin drama.

## Cómo proponer una Tech Talk

1. Agregar el tema al backlog de Tech Talks (Notion, Jira o donde el equipo lo tenga)
2. Incluir: título, descripción de 2-3 líneas y tiempo estimado
3. El Tech Lead o EM confirma el slot en el calendario

No hace falta tener todo preparado para proponer. La propuesta es el compromiso de prepararlo.

## Formatos válidos

**Demo**: Mostrar algo que construiste o exploraste. El código habla.

**Exploración**: "Estuve investigando X, esto es lo que encontré, esto es lo que no entiendo todavía."

**Problema y solución**: "Tuvimos este problema, lo resolvimos así, esto aprendimos."

**Debate técnico**: "Hay dos formas de hacer X, estas son las trade-offs, ¿qué piensa el equipo?"

**Postmortem técnico**: Compartir un incidente o bug complejo con el equipo para que todos aprendan.

## Qué no es una Tech Talk

- Una reunión de status
- Una presentación de producto
- Una capacitación obligatoria

## Registro

Las Tech Talks se graban o se documenta un resumen con los puntos clave. Eso permite que quien no pudo asistir acceda al contenido y que el equipo tenga un historial de lo que se exploró.
